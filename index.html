          ❓ SOP Token Reference Guide
          All syntax tokens available in the CHECKLIST editor
          QA CHECKLIST TOKENS
          # Section Name
          Section Header
          Bold green divider — structural only, no checkbox. Groups related QA items.
          Ex: # Gather SOULZ Parts
          > Sub-item
          Indented Sub-Check
          Smaller indented checkbox for secondary verifications under a main check.
          Ex: > Verify LED polarity visually
          Plain text
          or: - Plain text
          Standard QA Checkbox
          Any plain line (with or without a leading -) becomes a bold full-width checkbox. All must be checked before sign-off is allowed.
          [INPUT] Label
          Standalone Input Field
          Creates a labeled text entry row. Packer must fill it in. Value is logged in the QA audit record for traceability.
          Ex: [INPUT] Enter Unit Serial No.
          > Set power to [INPUT] W
          Inline Input (inside any line type)
          [INPUT] can appear inside any line — plain checkbox, sub-check >, or header #. Embeds a short text field inline for capturing measured values.
          Ex: > Confirm rated voltage is [INPUT] V
          [SCAN:Item Name]
          Physical Bin Scan NEW
          Renders a scan row with a mini QR code showing the bin label code (NGZ-ITEMNAME). Packer must point phone camera at the matching bin — validated in real-time. Blocks sign-off until confirmed.
          Format: Item name is slugified → NGZ-SK8LYTZ-UNIT
          Spaces → hyphens, always uppercase
          Ex: [SCAN:SK8Lytz Unit] validates bin labelled NGZ-SK8LYTZ-UNIT
          MEDIA TOKENS
          [IMG:https://...]
          Embed Image
          Renders a full-width image from a public URL. Use the 📎 MEDIA button to browse your Supabase bucket and insert the token automatically. Supports jpg, png, gif, webp, svg.
          [BARCODE:value]
          Code-128 Barcode
          Generates a scannable Code-128 linear barcode graphic. Display only — for active camera verification use [SCAN:] instead.
          Ex: [BARCODE:NGZ-SOULZ-BOX]
          [QR:value]
          QR Code
          Generates a QR code from any string or URL. Useful for linking to setup guides or instruction videos. Display only.
          Ex: [QR:https://zengge.app]
          FULL EXAMPLE
          # Gather SOULZ Parts
          2 x SK8Lytz Units
          [SCAN:SK8Lytz Unit] ← camera validates NGZ-SK8LYTZ-UNIT bin label
          [INPUT] Enter Unit Serial No.
          > Verify LED polarity before mounting
          > Confirm draw is [INPUT] mA at max brightness
          [IMG:https://your-bucket.supabase.co/sops/wiring.jpg]
          # QC & Pack
          Apply QC sticker to each unit
          [BARCODE:NGZ-SOULZ-BOX-STICKER]
